# Fase de Produccion Controlada del proyecto Radar

Objetivo:

- formalizar una fase de produccion controlada sin declarar un modelo final unico
- preservar la dualidad funcional entre `E1_v5_clean` y `E9_v2_clean`
- separar capa experimental y capa operativa controlada
- dejar congelada la infraestructura madura y la politica de promocion futura

Puntos canónicos:

- `E1_v5_clean` = referente numerico puro principal
- `E9_v2_clean` = referente operativo de riesgo-direccion-caidas
- `E10_v1_clean` = modelado inicial fragil, ya corrido pero no competitivo
- `E11` = familia futura dual numerica + categorica, todavia no ejecutada
- `C1` = corrida real y pausa temprana
- `C2-C4` = infraestructura preparada sin corrida
- explicabilidad transversal = parcial intra-familia

Entregables esperados:

- documento rector de fase
- separacion explicita experimental vs operativa
- benchmarks operativos vigentes formalizados
- politica de promocion futura
- actualizacion de plan, bitacora, README tecnico y artefactos canónicos derivados
