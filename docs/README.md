# docs

Estatus: `documental`

Proposito:

- documentar arquitectura, metodologia operativa y migraciones del repositorio

Subarboles:

- `architecture/`: estructura canonica, auditorias y decisiones de diseño del repo
- `operations/`: guias operativas y transiciones de uso
- `migration/`: tablas y notas de cambio estructural

Lo que si vive aqui:

- explicaciones de estructura y gobierno del repo
- decisiones de migracion y compatibilidad
- documentacion que un tercero necesita para entender el proyecto

Lo que no debe vivir aqui:

- codigo ejecutable
- datasets
- logs o runs

Relacion con otras carpetas:

- fija la lectura oficial de `src/`, `data/`, `artifacts/`, `experiments/` y `legacy/`
